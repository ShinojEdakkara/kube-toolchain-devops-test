"Rasio n/m" berarti bahwa Pemegang Lisensi menerima sejumlah ('n')
kepemilikan untuk Program Pendukung untuk setiap jumlah ('m') kepemilikan yang ditetapkan dari Program Utama secara keseluruhan. Kecuali apabila ditetapkan secara lain, jumlah kepemilikan untuk Program Pendukung dibulatkan hingga kelipatan 'n'. Sebagai contoh, apabila Program mencakup 100 PVU untuk sebuah Program Pendukung untuk setiap 500 PVU yang didapatkan dari Program Utama dan Pemegang Lisensi memperoleh 1.200 PVU Program, Pemegang Lisensi dapat memasang Program Pendukung dan memiliki inti prosesor yang tersedia untuknya atau dikelola olehnya hingga 300 PVU. PVU tersebut tidak perlu diperhitungkan sebagai bagian dari total persyaratan PVU untuk pemasangan Program oleh Pemegang Lisensi untuk
kepentingan pemasangan Program Pendukung (meskipun PVU tersebut mungkin perlu diperhitungkan karena alasan lainnya, seperti inti prosesor yang disediakan untuk Program Utama juga).

"Penggunaan berdasarkan Program Utama" berarti bahwa Program Pendukung diberikan secara eksklusif untuk digunakan berdasarkan Program Utama. Tidak ada Pemegang Lisensi atau aplikasi, program, atau alat apa pun yang berhak menggunakan atau mengakses layanan Program Pendukung secara langsung, kecuali Pemegang Lisensi dapat mengakses Program Pendukung guna melaksanakan fungsi administratif untuk Program Pendukung seperti pencadangan, pemulihan, dan konfigurasi yang sah.

"Panggilan API Bulanan" adalah unit ukuran yang olehnya Program Pendukung dapat diberi lisensi. Suatu Panggilan API adalah pemanggilan Program melalui antarmuka programatik. Kepemilikan yang memadai harus diperoleh untuk mencakup jumlah maksimum Panggilan API dalam bulan kalender mana pun.

Unit Nilai Prosesor (PVU) adalah unit ukuran yang melaluinya Program dapat diberi lisensi. Jumlah kepemilikan PVU yang diperlukan didasarkan pada teknologi prosesor (ditentukan dalam Tabel PVU berdasarkan Vendor, Merek, Jenis, dan Nomor Model Prosesor di http://www.ibm.com/software/lotus/passportadvantage/pvu_licensing_for_customers.html) dan jumlah prosesor yang disediakan untuk Program. IBM terus menentukan prosesor, untuk tujuan pemberian lisensi berdasarkan PVU, menjadi masing-masing inti prosesor pada sebuah keping. Sebagai contoh, keping dua inti (dual-core) prosesor memiliki dua inti prosesor.

* Inti prosesor yang diaktifkan adalah inti prosesor yang tersedia untuk digunakan dalam sebuah server fisik atau maya, tanpa memperhatikan apakah kapasitas inti prosesor dapat dibatasi atau dibatasi melalui teknologi-teknologi virtualisasi, perintah-perintah sistem operasi, pengaturan-pengaturan BIOS atau pembatasan-pembatasan yang sama.
